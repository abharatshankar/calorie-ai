# 📋 Calorie AI - Requirements Specification

Version: 1.0

---

# 1. Functional Requirements

## Authentication

### User Registration
- Users can create an account using Email and Password.
- User information will be securely stored.
- Duplicate email registration is not allowed.

### User Login
- Users can log in using registered credentials.
- JWT Authentication will be used.
- Users remain logged in until they explicitly log out.

### Guest Mode
- Users can use the application without signing in.
- Guest users can analyze food images.
- Guest users cannot save history.
- Guest users cannot access analytics or previous scans.

---

# 2. Food Analysis

The application shall allow users to:

- Capture a food image using the camera.
- Upload an image from the gallery.
- Send the image to the Vision AI service.
- Receive structured food analysis.

The AI response should include:

- Food Name
- Estimated Portion Size
- Estimated Calories
- Protein (g)
- Carbohydrates (g)
- Fat (g)
- Confidence Score (Future)

---

# 3. Food History

Authenticated users shall be able to:

- Save every food scan.
- View daily food history.
- View previous dates.
- Delete food entries.
- Refresh history.

Each history record should contain:

- Image
- Food Name
- Date
- Time
- Calories
- Protein
- Carbs
- Fat

---

# 4. Daily Summary

Users should be able to see:

- Total Calories Consumed
- Total Protein
- Total Carbohydrates
- Total Fat

Grouped by date.

---

# 5. AI Nutrition Insights

After every successful scan, AI may provide:

Examples:

"You have consumed a high amount of carbohydrates today."

"Consider adding more protein."

"Your calorie intake is within today's goal."

(MVP: Static prompt generated by LLM)

---

# 6. User Profile

Users can:

- View Profile
- Update Name
- Logout

Future:

- Height
- Weight
- Age
- Gender
- Activity Level
- Goal Weight

---

# 7. Error Handling

The application should gracefully handle:

- No Internet
- AI API Failure
- Invalid Image
- Empty Response
- Authentication Failure
- Server Errors

Every error should display a user-friendly message.

---

# 8. Non-Functional Requirements

## Performance

- Food analysis should complete within 5–8 seconds.
- API response time should be under 2 seconds (excluding AI processing).
- History should load in under 1 second.

---

## Security

- JWT Authentication
- Password Hashing
- HTTPS
- Secure Image Upload
- Input Validation

---

## Scalability

The backend should support:

- Multiple concurrent users
- Future AI provider replacement
- Horizontal scaling

---

## Maintainability

Architecture should follow:

Frontend

- Clean Architecture
- Repository Pattern
- Riverpod

Backend

- Layered Architecture
- Service Layer
- Repository Layer
- Dependency Injection

---

# 9. User Stories

## Guest User

As a guest,

I want to upload a food image

so that I can quickly know the calorie information without creating an account.

---

## Registered User

As a registered user,

I want my food scans to be saved,

so that I can track my eating habits.

---

## Health Conscious User

As a fitness enthusiast,

I want to know protein, carbohydrates and fat,

so that I can maintain my nutrition goals.

---

## Returning User

As a returning user,

I want to see my previous meals,

so that I can compare my daily calorie intake.

# 11. Success Criteria

The MVP will be considered successful if:

- A user can scan a meal in under one minute.
- AI identifies common food items with reasonable accuracy.
- Logged-in users can view their meal history.
- Guest users can analyze food without signing in.
- The application is successfully deployed and accessible online.
